## 🗓️ Week-by-Week Breakdown

---

### 📌 Week 1 – HTML Foundations
**Phase:** Beginner

The starting point of everything. This week was all about understanding how the web is structured and getting comfortable writing HTML from scratch.

**What I learned:**
- What HTML is and why it's the backbone of every web page
- The basic structure of an HTML document: `<!DOCTYPE>`, `<html>`, `<head>`, `<body>`
- Text elements: headings, paragraphs, bold and italic formatting
- Links (`<a>`) and images (`<img>`) — how to connect pages and embed media correctly
- Lists: unordered (`<ul>`), ordered (`<ol>`), and tables for organizing data
- Introduction to forms: inputs, labels, and buttons

**Mindset:** No rushing. Understanding every line before moving forward.

---

### 🎨 Week 2 – CSS Styling
**Phase:** Beginner

With HTML solid, this week brought in CSS — the layer that makes everything look intentional and polished.

**What I learned:**
- What CSS is and how it works alongside HTML
- Changing fonts: size, family, weight, and line spacing
- Adding colors: background colors, text colors, and picking combinations that work visually
- Creating hover effects on buttons and links for interactivity
- Combining HTML structure with CSS styling to produce a complete, well-designed first app

**Milestone:** Built my **first full app** combining everything from Weeks 1 and 2.

---

### 🦁 Week 3 – Pine City Zoo App
**Phase:** Beginner → Intermediate transition

This week moved from pure learning into real-world thinking and building. Using the **Software Development Life Cycle (SDLC)** as a guide, I built my own version of the Pine City Zoo app.

**What I learned:**
- The Software Development Life Cycle (SDLC): planning, designing, building, testing
- How to think like a developer before writing a single line of code
- Breaking a project into sections and tackling it piece by piece
- Debugging: figuring out what went wrong and why

**Project:** Pine City Zoo — a multi-section website with structured HTML and fully styled CSS, built independently.

---

### ⚡ Week 4 – Introduction to JavaScript
**Phase:** Intermediate

The biggest shift so far. After two static languages (HTML and CSS), JavaScript introduced logic, interactivity, and dynamic behavior.

**What I learned:**
- Why static apps are limited and what JavaScript solves
- JavaScript as the third pillar of web development alongside HTML and CSS
- Core JS concepts: variables, functions, events, and DOM manipulation
- How to make a page react to user input
- Performing calculations, showing/hiding content, and storing user input

**Mindset:** Be patient. A new way of thinking takes time to click.

---

### 🧮 Week 5 – Calculator App
**Phase:** Intermediate

The first real JavaScript-powered project — a fully functional calculator built from scratch.

**What I learned (applied):**
- HTML for the calculator layout: display screen and buttons
- CSS to make it look clean and user-friendly
- JavaScript for all the logic: button click events, number input, and arithmetic operations

**Why it mattered:** This was the first time an app I built could actually respond to users and perform real functionality. It brought all three core skills together for the first time.

**Project:** A working browser-based calculator.

---

### 📇 Week 6 – Contact Book App & APIs
**Phase:** Intermediate (Final Week)

The most advanced project yet — a Contact Book app that connects to a real external API to pull live data.

**What I learned:**
- What an API (Application Programming Interface) is and why it matters
- How to make HTTP requests from JavaScript using `fetch()`
- Working with JSON data: parsing it, handling responses, and rendering it into HTML
- Understanding dynamic data — apps that respond to real-world information
- How modern apps pull live weather, user profiles, social feeds, and database records

**Project:** A fully functional Contact Book that fetches real contact data from an external API and displays it dynamically in the browser.

**Milestone:** End of the Intermediate Phase — I could now build apps that talk to the internet.

---

### 🐍 Week 7 – Introduction to Python
**Phase:** Advanced

A fresh start with a brand new language. Stepping away from the browser entirely and into backend development with Python.

**What I learned:**
- Python syntax and structure vs. JavaScript
- How prior programming knowledge transfers to a new language
- Variables, data types, conditionals, loops, and functions in Python
- The foundations of backend development and building APIs with Python

**Mindset:** Starting from scratch again, but not from zero — everything learned in previous weeks accelerated the process.

---

### 🏗️ Week 8 – Python Data Structures & Backend Project
**Phase:** Advanced

Going deeper into Python with a focus on data structures and building a complete backend system.

**What I learned:**
- **Lists** – ordered, mutable collections
- **Dictionaries** – key-value pairs for structured data
- **Tuples** – ordered, immutable collections
- **Sets** – unordered collections with no duplicates
- How to design backend architecture from scratch
- Writing, testing, and debugging a Python backend project independently

**Project:** A complete Python backend application built from the ground up — architecture designed, coded, tested, and debugged entirely by me.

---

### 🏁 Week 9 – Final Push & Wrap-Up
**Phase:** Advanced (Final Week)

The last week of the FNB App Academy. Not the end — the beginning of what comes next.

**What I did:**
- Reflected on 9 weeks of growth across 3 phases and 3 languages
- Put the finishing touches on all projects
- Identified and strengthened any remaining weak spots
- Consolidated everything learned into a final Python project
- Prepared to take these skills into real-world development

**What I achieved across the full course:**
- Built real, functional projects — not demos
- Learned HTML, CSS, JavaScript, and Python
- Went from static web pages to API-connected apps to backend systems
- Developed a developer's mindset: plan, build, test, debug, improve

## 🚀 How to Run the Projects

**For HTML/CSS/JS projects (Weeks 1–6):**
1. Open the relevant week's folder
2. Open `index.html` in any modern browser
3. No installation required

**For Python projects (Weeks 7–9):**
1. Make sure Python 3 is installed: `python --version`
2. Navigate to the relevant week's folder
3. Run: `python main.py`
